## üêõ Troubleshooting

### Common Issues & Solutions

#### Issue: "Port 8000 already in use"
```bash
# Find process using port 8000
lsof -i :8000  # macOS/Linux
netstat -ano | findstr :8000  # Windows

# Kill the process
kill -9 <PID>  # macOS/Linux
taskkill /PID <PID> /F  # Windows

# Use different port
python -m http.server 8001
```

#### Issue: "npm install fails"
```bash
# Clear npm cache
npm cache clean --force

# Delete node_modules and package-lock.json
rm -rf node_modules package-lock.json

# Reinstall
npm install
```

#### Issue: Games not loading
- Check browser console (F12) for errors
- Ensure all file paths are correct
- Clear browser cache (Ctrl+Shift+Delete or Cmd+Shift+Delete)
- Try different browser

#### Issue: High scores not saving
- Check if localStorage is enabled
- Try incognito/private browsing mode
- Clear browser cookies and cache

## ‚ùì FAQ

**Q: Can I play these games offline?**
A: Yes! Most games work offline. Whack-a-Mole has PWA support with a service worker for true offline play.

**Q: How do I customize a game?**
A: Each game is self-contained. Modify the CSS files for styling, JS files for mechanics, and HTML for structure.

**Q: Can I use these games commercially?**
A: Yes, within the MIT License terms. Attribute the original author and maintain the license.

**Q: Why aren't my high scores saving?**
A: LocalStorage requires HTTPS in production or HTTP on localhost. Check browser console for errors.

**Q: How do I contribute to this project?**

**Q: Can I add new games?**

**Q: Is this project optimized for mobile?**

**Q: What if I find a bug?**
